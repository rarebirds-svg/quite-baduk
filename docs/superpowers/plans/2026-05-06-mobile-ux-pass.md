# 모바일 UX 종합 패스 (2026-05-06)

## 배경

사용자가 모바일 브라우저에서 접속한 스크린샷에서 TopNav의 "K-Baduk" 텍스트가 두 줄로 깨져 있는 등 모바일 폭(360–414px)에서 다수의 레이아웃·가독성 이슈가 보고됨. 종합 모바일 패스로 주요 화면을 일괄 점검·수정한다.

## 목표

`360 / 390 / 414` 폭 (iPhone SE3 / iPhone 14 / iPhone 14 Pro Max 평균대) 에서 모든 1차·2차 화면이 다음을 만족.

- 가로 스크롤 없음 (사용자가 의도적으로 가로 스크롤하는 표 제외)
- 모든 텍스트가 잘리지 않고 읽힘 (헤딩 wrap 허용)
- 모든 인터랙션 요소 터치 타깃 ≥ 40×40 px
- TopNav 1열 내 정리, 브랜드 텍스트 줄바꿈 없음
- 다이얼로그·시트가 모바일 폭에 맞게 fit
- 주요 테이블은 반응형 카드·요약뷰로 대체 또는 가로 스크롤 컨테이너 명시

## 범위

### 1차 (대중 사용)
- TopNav (글로벌)
- `/` 닉네임 게이트 / 랜딩
- `/game/new` 새 대국
- `/game/play/[id]` 대국 진행
- `/game/review/[id]` 기보 복기
- `/daily` 일일 챌린지

### 2차 (보조)
- `/history` 전적
- `/settings` 설정

### 3차 (관리자, 모바일 우선순위 낮음 — 안 깨지기만 보장)
- `/admin`, `/admin/login-history`, `/admin/sessions/[id]`

### 범위 외
- `/privacy`, `/terms` (정적 법문서, 대체로 OK)
- `/dev/components` (내부 갤러리)

## 전략

각 화면을 다음 5 항목으로 점검.

1. 컨테이너 폭. `max-w-*` 와 `px-*` 가 좁은 폭에서 의미 있는 여백을 만드는지.
2. 텍스트 스케일. 헤딩 (`text-3xl` 이상) 이 wrap 자연스럽게 되는지, 본문 가독성.
3. 인터랙션. 버튼·아이콘 버튼이 ≥40×40, 인접 타깃 간 충분한 간격.
4. 테이블·리스트. 가로 스크롤 / 카드 변형 / 칼럼 숨김 중 선택.
5. 다이얼로그·시트·드롭다운. 모바일에서 풀폭 또는 적절한 패딩.

## 진행 단계

수정은 클러스터별 커밋으로 나눠 롤백 단위를 명확히 한다.

1. **TopNav** — 즉각적 시각 깨짐. 우선 처리.
2. **게임 흐름** — new → play → review (보드와 컨트롤). 핵심 사용 흐름.
3. **일일 챌린지 / 전적** — 보조 사용 흐름.
4. **설정 / 관리자** — 안 깨짐 확보 수준.
5. **정리** — 새로 등장한 토큰·유틸 클래스가 있으면 디자인 시스템 규칙 정합성 점검.

## 검증

- 코드 변경 후 `npm run type-check` + `npm run lint` + `npm test -- --run`.
- 라이브 dev 서버를 모바일 에뮬레이션 (Chrome DevTools 또는 Playwright headless) 으로 360 / 390 / 414 폭 스냅샷 확인.
- 사용자 실제 단말 (Galaxy, SKT) 재확인 요청.

## 성공 기준

- 1차·2차 화면이 360px 폭에서 가로 스크롤 없이 모든 핵심 액션이 한 화면 안에서 도달 가능.
- TopNav 브랜드 wrap 없음, 우측 항목이 좁은 폭에서 자동 정리됨 (드롭다운 흡수 등).
- 기존 데스크탑 레이아웃은 동일하게 유지.

## 범위 외 / 후속

- 본격적 모바일 PWA 인스톨 흐름 다듬기 (manifest 아이콘 등은 이미 존재).
- 홈 히어로 카피 "바둑, 조용한 승부" → 새 브랜드 정합 (이번 패스와 별개).
- 다국어 카피 회귀 (한국어 콜론 종결 — 별건 작업).
